Cost and pricing

It costs a few dollars a month to serve a worker.

The biggest cost isn't messaging — it's AI. That's worth knowing, because AI cost is something we control directly through model choice and caching. The result is a comfortable margin and room to lower the price as the user base grows.

Where the cost actually goes

You'd expect messaging to be the main cost. It isn't, as long as replies stay inside WhatsApp's free 24-hour window — which is how a normal conversation runs anyway. With messaging mostly free, the main cost is AI, and that varies a lot depending on one thing: which model does the routine work.

Because AI is the swing variable and we control it, cost is a lever we hold, not an external rate we're at the mercy of.

The hidden line item the headline misses

Messaging numbers aren't zero-churn — some get banned and need warming/replacing. Budget a notional $1–3/worker/month ban-recovery reserve on top of the raw cost. We'd rather over-budget this than be surprised by it.

The cost levers

Stay in the free window

Keep replies inside the 24-hour window and messaging collapses to marketing-only — the single biggest lever.

Right-size the model

A fast, cheap model for routine extraction; a stronger model only for drafting and judgement.

Cache the prompt

One large cached system prompt across every message saves ~90% on the repeated portion.

Batch the backfill

History backfill runs through the discounted Batch API at half price.

Two scenarios, priced on the worse one

Messaging cost depends a lot on how a worker uses the channel. Two scenarios bracket it, and the sensible thing is to set the price floor against the worse case:

Window-heavy

Conversation-led, mostly inside the free window. Messaging cost ~$0.50/worker. The common case.

Marketing-heavy

Lots of out-of-window re-engagement via paid templates. Messaging cost ~$5–15/worker. Price the floor here.

Re-verify before committing the model

Two inputs move the whole model and must be re-checked before any pricing commitment: AI pricing (the research cache is dated 26 May 2026) and Meta's per-country message rates (revised roughly every 6 months, and they vary 10–60× by destination).

Tech info: model routing & messaging-partner economics For technical readers
  • Model routing: a fast, low-cost model handles per-message extraction/classification/flagging; a stronger model is reserved for drafting and judgement. One large cached system prompt across every inbound message (~90% off the cached portion); Batch API (−50%) for history backfill.
  • Messaging-partner choice matters: flat-fee-plus-per-message partners vs send-and-receive-per-message pricing vs percentage markups produce materially different bills at scale. Model the partner, not just Meta's card.
  • Always-on floor: the worker process + Redis must be on paid hosting (free database projects pause after ~7 days idle) — a fixed cost the per-worker number amortises as volume grows.
