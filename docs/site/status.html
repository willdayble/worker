Open questions & blockers

Formal user testing hasn't started yet — that's gated on getting the database live — so these aren't bugs found by testers. They're the known questions and blockers that could slow or reshape the build. None of them is a dead-end.

High

WhatsApp Coexistence may not be available in Australia

The durable official WhatsApp path is commonly switched off in AU (and the UK/EU). If it's unavailable for the real user's region, that lane won't work there and we lean on Telegram plus the official path in regions that do support it. It needs a live check.

Open question · checked as part of the kill-test's official lane · shapes the channel strategy for AU users, not the product as a whole.

High

The database isn't live yet

The schema and access rules are written but not yet applied to the live database (a network snag), and the encryption master key isn't in the secrets manager. Until both are done, nothing runs end-to-end against real data — so real test users can't onboard. Both are ops steps, not redesigns.

Blocker for the next phase · solvable via the Supabase dashboard + one secret to add · this is the main thing standing between "built" and "first real test user".

Medium

Unofficial WhatsApp survivability is still unverified

The two-to-four-week ban test needs a person with a throwaway SIM, and it hasn't been run, so we don't yet know if the Baileys fallback is worth keeping. This only affects the fallback — Telegram and the official path don't depend on it.

Not dev-stopping · the harness is built and ready to run.

Medium

The channel worker isn't deployed yet

It's built and its tests pass on their own, but it hasn't been committed, deployed, or wired to the live database, so the inbox isn't receiving real messages yet. It's waiting on the database going live (above).

Integration step · follows directly from the database blocker.

Good

No fatal problems so far

Worth saying plainly: nothing found so far stops the build. The design is deliberately decoupled, so even the worst case — a WhatsApp number getting banned — costs a channel, not the client list, the safety flags, or the bookings. The current blockers are setup and availability steps, not architectural dead-ends.
